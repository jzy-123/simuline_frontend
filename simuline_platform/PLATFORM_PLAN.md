# SimuLine 平台化规划

## 目标定位

把当前 SimuLine 科研代码包装成一个“仿真实验与策略评估平台”。客户不需要直接运行 Python 脚本或编辑 Excel，而是通过网页选择业务场景、配置策略、运行仿真、查看指标曲线和策略报告。

## 第一阶段：结果展示 Demo

目标：最快形成可给客户看的平台界面。

已完成：

- 新增 `simuline_platform` 包
- 新增 FastAPI 应用入口
- 自动扫描 `Out/Result/{experiment}/{variant}/{run}_output.csv`
- 提供实验列表、实验详情、指标分组和报告 API
- 新增静态仪表盘页面
- 读取已有 `Rebuttal / promote-creator / run1` 输出并展示

验收标准：

- 能启动 Web 服务
- 能打开首页
- 能看到已有实验
- 能看到指标曲线和业务解读

## 第二阶段：实验配置与任务提交

目标：客户可以从页面发起小规模仿真实验。

已完成：

- 增加实验模板选择页面
- 将核心参数映射为业务表单
- 用 Pydantic 校验页面输入
- 封装 `run_simulation(config)`，摆脱 Excel 强依赖
- 建立实验工作目录：`workspace/platform_jobs/{job_id}`
- 使用独立 Python 子进程执行仿真任务
- 记录任务状态：pending / starting / running / completed / failed
- 通过输出 CSV 轮询当前仿真轮次，形成进度条

待做：

- 增加账号登录
- 增加任务取消
- 增加并发限制
- 接入 Redis/Celery 或 RQ，替换当前单机子进程调度

建议第一批模板：

- 基础个性化推荐
- 新内容扶持
- 热门内容倾斜
- 作者/供给方推广

## 第三阶段：策略对比与报告

目标：从“看结果”升级到“辅助决策”。

待做：

- 多实验指标对比
- 策略 A/B 分析
- 指标变化解释规则
- 自动生成业务报告
- 导出 PDF / HTML 报告
- 支持复制实验配置

## 第四阶段：客户业务适配

目标：把新闻推荐语义映射到客户自己的业务。

待做：

- 将 `article` 抽象为内容、商品、营销素材或资讯
- 将 `creator` 抽象为品牌、供应商、业务单元或内容生产方
- 将 like/click/exposure 映射为客户业务指标
- 支持客户历史数据导入
- 支持定制推荐策略
- 支持更多业务约束和目标函数

## 工程优先级

1. 保持核心仿真代码可独立运行。
2. 平台层只做包装、配置、调度和展示。
3. 先展示已有结果，再接入新实验运行。
4. 先支持单机单用户试用，再扩展多用户和队列。
5. 先用小规模 demo 配置，避免客户试用时任务过慢。
